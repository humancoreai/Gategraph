# Release v0.9.2_CANDIDATE

## Phase

Multi-Agent / Multi-Mode Architecture Definition

## Base

v0.9.1_STABLE

## Nature of Change

Architecture and boundary definition only.

No autonomous agents, distributed governance, new runtime execution model, new risk model, self-orchestration, or governance write authority are introduced.

## Acceptance Focus

- Agent means governed runtime identity, not governance participant.
- Mode means narrowing constraint profile, not authority expansion.
- Delegation means audited parent-child task lineage, not autonomous orchestration.
- Budget authority remains central.
- Replay requires complete audit-visible causal lineage.
- Emergence paths are explicitly forbidden.
