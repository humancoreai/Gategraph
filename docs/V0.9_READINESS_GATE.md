# v0.9 Readiness Gate

Purpose: verify that the v0.8 line is internally consistent enough to be considered for a v0.9 milestone.

This gate is descriptive and structural. It does not introduce new governance behavior.

Checks:
- release metadata is present
- repository entry points are present
- evidence manifest contains late-stage archive, replay, drift and export evidence
- generated runtime artifacts are not required for source release

Out of scope:
- new governance features
- policy tuning
- automatic recommendations
- risk scoring
- prediction
